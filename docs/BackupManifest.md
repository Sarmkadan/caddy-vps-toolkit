# BackupManifest

The `BackupManifest` class represents metadata and configuration data for a backup snapshot generated by the Caddy VPS toolkit. It encapsulates essential details about the backup, including its unique identifier, creation timestamp, associated services, and configuration values, enabling restoration or inspection of the backup state.

## API

### `public string BackupId`
Unique identifier for the backup. This value is generated at creation time and serves as the primary key for referencing the backup in storage or restoration operations.

### `public DateTime CreatedAt`
Timestamp indicating when the backup was created. This value is set automatically during instantiation and reflects the local system time at creation.

### `public string AppVersion`
Version of the application (Caddy VPS toolkit) that generated the backup. This field helps ensure compatibility during restoration by identifying the tooling version used to create the backup.

### `public List<ManagedService> Services`
List of services managed by the Caddy instance at the time of backup. Each `ManagedService` entry includes service-specific configuration and state, enabling full restoration of service definitions.

### `public Dictionary<string, string> Configuration`
Key-value pairs representing the global configuration of the Caddy instance at the time of backup. This dictionary includes environment variables, runtime settings, and other non-service-specific configurations.

### `public string? CaddyfileContent`
Optional raw content of the `Caddyfile` used by the Caddy instance at the time of backup. This field may be `null` if the backup was created without capturing the `Caddyfile` content.

### `public string Description`
Human-readable description of the backup. This field allows users to annotate backups with contextual information (e.g., "Pre-upgrade snapshot" or "Stable configuration").
