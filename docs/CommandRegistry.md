# CommandRegistry

`CommandRegistry` is a container that stores and retrieves command definitions for a CLI tool. It maps command names to `CommandDescriptor` instances, provides lookup and enumeration methods, and can generate formatted help text describing all registered commands. The type serves as the central catalogue of available operations in the `caddy-vps-toolkit` application.

## API

### `public void Register`

Registers a `CommandDescriptor` under its associated command name. If a command with the same name already exists, the method overwrites the previous entry.

| Parameter | Type | Description |
|---|---|---|
| `descriptor` | `CommandDescriptor` | The command definition to store. Its `Name` property is used as the key. |

**Returns:** Nothing.

**Throws:** `ArgumentNullException` when `descriptor` is `null`.

---

### `public CommandDescriptor Get`

Retrieves a registered command by name.

| Parameter | Type | Description |
|---|---|---|
| `name` | `string` | The case-sensitive command name to look up. |

**Returns:** The matching `CommandDescriptor`, or `null` if no command with that name is registered.

**Throws:** `ArgumentNullException` when `name` is `null`.

---

### `public bool Exists`

Checks whether a command with the given name has been registered.

| Parameter | Type | Description |
|---|---|---|
| `name` | `string` | The case-sensitive command name to test. |

**Returns:** `true` if the command exists; `false` otherwise.

**Throws:** `ArgumentNullException` when `name` is `null`.

---

### `public List<CommandDescriptor> GetAll`

Returns a flat list of every `CommandDescriptor` currently stored in the registry. The order reflects insertion order, with overwritten entries occupying the position of the original.

**Returns:** A `List<CommandDescriptor>` containing all registered commands. Returns an empty list when no commands are registered.

---

### `public string GenerateHelpText`

Produces a formatted string summarising all registered commands, suitable for display as top-level help output. The format includes the registry's own `Name`, `Description`, and `Usage`, followed by a listing of each command with its usage line and description.

**Returns:** A multi-line `string` containing the help text.

---

### `public string Name`

Gets or sets the name of the command set (e.g. the application or tool name). This value appears in the header of the help text generated by `GenerateHelpText`.

---

### `public string Description`

Gets or sets a short description of the command set. Included in the help text header.

---

### `public string Usage`

Gets or sets the general usage pattern for invoking the tool (e.g. `tool <command> [options]`). Included in the help text header.

---

### `public List<string> RequiredArguments`

Gets the list of positional argument names that are required when invoking the command. This property is exposed on `CommandDescriptor` instances stored in the registry.

---

### `public List<string> OptionalFlags`

Gets the list of optional flag names (typically prefixed with `--` or `-`) that the command accepts. This property is exposed on `CommandDescriptor` instances stored in the registry.

---

### `public CommandDescriptor`

Constructor. Creates a new `CommandDescriptor` with the specified name and description. The instance starts with empty required-arguments and optional-flags lists.

| Parameter | Type | Description |
|---|---|---|
| `name` | `string` | The command name used for registration and lookup. |
| `description` | `string` | A human-readable summary of what the command does. |

**Throws:** `ArgumentNullException` when `name` or `description` is `null`.

---

### `public CommandDescriptor WithUsage`

Returns a new `CommandDescriptor` that is a copy of the current instance but with the specified usage string substituted. The original instance is not modified.

| Parameter | Type | Description |
|---|---|---|
| `usage` | `string` | The usage pattern string (e.g. `deploy <site> [--force]`). |

**Returns:** A new `CommandDescriptor` with the given usage.

**Throws:** `ArgumentNullException` when `usage` is `null`.

---

### `public CommandDescriptor RequireArgument`

Returns a new `CommandDescriptor` that is a copy of the current instance with the given argument name appended to its `RequiredArguments` list. The original instance is not modified.

| Parameter | Type | Description |
|---|---|---|
| `argumentName` | `string` | The name of the required positional argument. |

**Returns:** A new `CommandDescriptor` with the additional required argument.

**Throws:** `ArgumentNullException` when `argumentName` is `null`.

---

### `public CommandDescriptor AllowFlag`

Returns a new `CommandDescriptor` that is a copy of the current instance with the given flag name appended to its `OptionalFlags` list. The original instance is not modified.

| Parameter | Type | Description |
|---|---|---|
| `flagName` | `string` | The name of the optional flag (e.g. `--verbose`). |

**Returns:** A new `CommandDescriptor` with the additional optional flag.

**Throws:** `ArgumentNullException` when `flagName` is `null`.

## Usage

### Example 1: Building a registry and printing help

```csharp
var registry = new CommandRegistry
{
    Name = "caddy-vps",
    Description = "Manage Caddy on a VPS",
    Usage = "caddy-vps <command> [options]"
};

var deployCmd = new CommandDescriptor("deploy", "Deploy a site configuration")
    .WithUsage("deploy <site-name> [--force]")
    .RequireArgument("site-name")
    .AllowFlag("--force");

var statusCmd = new CommandDescriptor("status", "Show server status")
    .WithUsage("status [--json]")
    .AllowFlag("--json");

registry.Register(deployCmd);
registry.Register(statusCmd);

Console.WriteLine(registry.GenerateHelpText());
```

### Example 2: Dispatching based on user input

```csharp
string userCommand = args.Length > 0 ? args[0] : "";

if (!registry.Exists(userCommand))
{
    Console.WriteLine($"Unknown command: {userCommand}");
    Console.WriteLine(registry.GenerateHelpText());
    return;
}

var descriptor = registry.Get(userCommand);

// Validate required arguments are present
if (args.Length - 1 < descriptor.RequiredArguments.Count)
{
    Console.WriteLine($"Usage: {descriptor.Usage}");
    return;
}

// Execute command logic using descriptor and remaining args...
```

## Notes

- **Overwrite behaviour:** `Register` replaces any existing descriptor with the same name. No error or warning is raised for the overwrite. If preserving the original is required, call `Exists` before registering.
- **Immutability of descriptors:** `WithUsage`, `RequireArgument`, and `AllowFlag` all return new instances. The original `CommandDescriptor` is never mutated. This makes descriptors safe to share across threads once constructed, but the registry itself is mutable.
- **Thread safety:** The registry's internal storage is not synchronised. Concurrent calls to `Register`, `Get`, `Exists`, or `GetAll` from multiple threads without external locking may produce inconsistent results or cause enumeration exceptions.
- **Null handling:** All methods that accept reference-type parameters throw `ArgumentNullException` on `null` input. `Get` and `Exists` return `null`/`false` for genuinely absent names, not for `null` keys.
- **Help text format:** `GenerateHelpText` assumes all descriptors have non-null `Name`, `Description`, and `Usage`. If a descriptor was constructed with nulls bypassing the constructor guard (e.g. via deserialisation), the output may contain empty or null segments.
- **Case sensitivity:** Command names are treated case-sensitively. `Get("Deploy")` will not find a command registered as `"deploy"`.
