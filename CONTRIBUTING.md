# Contributing to caddy-vps-toolkit

We welcome contributions! Please follow these guidelines:

1. **Code Style**: 4-space indentation, follow C# naming conventions (PascalCase for public, camelCase for private).
2. **Workflow**: Create a feature branch (`feature/name` or `fix/name`), make changes, and run tests.
3. **Commit Messages**: Use clear messages (feat, fix, docs, refactor, chore).
4. **Pull Requests**: Submit a PR to `main` with a clear description of changes.
5. **Testing**: Ensure all tests pass (`dotnet test`) and code coverage is maintained.
6. **Documentation**: Update `README.md` and `docs/` for any user-facing changes.

By contributing, you agree that your contributions will be licensed under the MIT License.

Check out our [GitHub Discussions](https://github.com/sarmkadan/caddy-vps-toolkit/discussions) for questions or support.
